Core workflow

Assume role, then start session

aws sts assume-role \ --role-arn arn:aws:iam::225726002432:role/OrganizationAccountAccessRole \ --role-session-name zero-audit \ --region us-east-1 # export the temporary credentials you get back export AWS_ACCESS_KEY_ID=... export AWS_SECRET_ACCESS_KEY=... export AWS_SESSION_TOKEN=... # then start SSM session aws ssm start-session --target i-002d7492322ae7f18 --region us-east-1

Safe rule: once the role is assumed, do all Miguel AWS work through that temporary role session.
